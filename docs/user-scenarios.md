## S-01. 신규 가입 및 초기 설정

> **사용자 유형**: 신규 사용자 (소유자)  
> **시작 조건**: 마이링크 서비스에 아직 가입하지 않은 상태  
> **관련 기능**: F-01 (Google 소셜 로그인), F-02 (프로필 설정)

### 흐름

1. 사용자는 **마이링크 서비스를 알아보기 위해** 루트 페이지(`/`)에 접속한다.
2. 사용자는 **서비스에 가입하기 위해** "시작하기" 또는 "Google로 로그인" 버튼을 클릭한다.
3. 사용자는 **본인 인증을 하기 위해** Google OAuth 팝업에서 자신의 Google 계정을 선택한다.
4. 시스템은 **사용자 프로필을 초기화하기 위해** Firestore에 신규 문서를 자동 생성한다.
   - `displayName`: Gmail 주소의 `@` 앞부분을 추출하여 설정 (예: `yangdogyun0421`)
   - `username`: Google 계정의 표시 이름을 가져와 설정 (예: `양도균`)
   - `photoURL`: Google 프로필 사진 URL
5. 사용자는 **자신의 링크를 관리하기 위해** 대시보드(`/dashboard`) 페이지로 자동 리다이렉트된다.
6. 사용자는 **자신의 공개 페이지 URL을 확인하기 위해** 대시보드에 표시된 URL(`도메인/{displayName}`)을 확인한다.

### 완료 조건
- Firestore에 사용자 문서가 생성되어 있다.
- `usernames/{displayName}` 문서에 userId가 매핑되어 있다.
- 대시보드 페이지가 정상적으로 표시된다.

## S-02. 링크 추가

> **사용자 유형**: 소유자  
> **시작 조건**: 로그인 상태로 대시보드에 접속한 상태  
> **관련 기능**: F-03 (링크 CRUD), F-04 (링크 파비콘 아이콘)

### 흐름

1. 사용자는 **새 링크를 등록하기 위해** 대시보드에서 "링크 추가" 버튼을 클릭한다.
2. 사용자는 **링크 제목을 지정하기 위해** 제목 입력 필드에 텍스트를 입력한다. (예: `내 GitHub`)
3. 사용자는 **링크 URL을 지정하기 위해** URL 입력 필드에 주소를 입력한다. (예: `https://github.com/yangdogyun0421`)
4. 시스템은 **URL의 유효성을 확인하기 위해** http:// 또는 https://로 시작하는지 검증한다.
   - 유효하지 않은 경우 → 오류 메시지를 표시한다.
5. 사용자는 **링크를 저장하기 위해** "저장" 또는 확인 버튼을 클릭한다.
6. 시스템은 **링크 데이터를 저장하기 위해** Firestore의 `users/{userId}/links` 서브 컬렉션에 새 문서를 생성한다.
7. 시스템은 **링크의 시각적 구분을 위해** Google Favicon API(`https://www.google.com/s2/favicons?domain=github.com&sz=64`)를 통해 파비콘을 로드하여 표시한다.
8. 사용자는 **추가된 링크를 확인하기 위해** 대시보드의 링크 목록에서 새로 추가된 항목을 확인한다.

### 완료 조건
- 새 링크가 Firestore 서브 컬렉션에 저장되어 있다.
- 대시보드 링크 목록에 새 항목이 파비콘과 함께 표시된다.
- 공개 프로필 페이지에도 해당 링크가 노출된다.

## S-03. 링크 수정 (인라인 편집)

> **사용자 유형**: 소유자  
> **시작 조건**: 로그인 상태로 대시보드에 접속, 1개 이상의 링크가 등록된 상태  
> **관련 기능**: F-03 (링크 CRUD)

### 흐름

1. 사용자는 **링크 제목을 수정하기 위해** 대시보드 링크 목록에서 수정할 링크의 제목 텍스트를 클릭한다.
2. 시스템은 **인라인 편집 모드를 제공하기 위해** 해당 텍스트를 즉시 편집 가능한 입력 필드로 전환한다.
3. 사용자는 **새로운 제목을 입력하기 위해** 기존 텍스트를 수정한다.
4. 사용자는 **변경 사항을 확정하기 위해** 입력 필드 바깥을 클릭하거나 Enter 키를 누른다.
5. 시스템은 **수정 내용을 반영하기 위해** Firestore의 해당 링크 문서를 업데이트한다.
6. (URL 수정의 경우) 사용자는 **링크 URL을 수정하기 위해** URL 텍스트를 클릭하여 동일한 인라인 편집 과정을 반복한다.
7. (URL 변경 시) 시스템은 **URL 유효성을 확인하기 위해** 변경된 URL이 올바른 형식인지 검증한다.
8. (URL 변경 시) 시스템은 **파비콘을 갱신하기 위해** 새 도메인에 맞는 파비콘을 Google Favicon API로 다시 로드한다.

### 완료 조건
- Firestore의 링크 문서가 수정된 값으로 업데이트되어 있다.
- 대시보드에 수정된 내용이 즉시 반영되어 있다.
- (URL 변경 시) 파비콘이 새 도메인에 맞게 갱신되어 있다.

## S-04. 링크 삭제

> **사용자 유형**: 소유자  
> **시작 조건**: 로그인 상태로 대시보드에 접속, 1개 이상의 링크가 등록된 상태  
> **관련 기능**: F-03 (링크 CRUD)

### 흐름

1. 사용자는 **불필요한 링크를 제거하기 위해** 대시보드 링크 목록에서 삭제할 링크의 "삭제" 버튼을 클릭한다.
2. 시스템은 **실수로 인한 삭제를 방지하기 위해** "정말 삭제하시겠습니까?" 확인 다이얼로그를 표시한다.
3. 사용자는 **삭제를 확정하기 위해** 확인 다이얼로그에서 "삭제" 버튼을 클릭한다.
   - 취소를 클릭한 경우 → 삭제가 취소되고 원래 상태로 돌아간다.
4. 시스템은 **링크를 제거하기 위해** Firestore의 `users/{userId}/links/{linkId}` 문서를 삭제한다.
5. 시스템은 **삭제 결과를 반영하기 위해** 대시보드 링크 목록에서 해당 항목을 즉시 제거한다.

### 완료 조건
- Firestore에서 해당 링크 문서가 삭제되어 있다.
- 대시보드 링크 목록에서 해당 항목이 사라져 있다.
- 공개 프로필 페이지에서도 해당 링크가 더 이상 표시되지 않는다.

## S-05. 프로필 수정 (인라인 편집)

> **사용자 유형**: 소유자  
> **시작 조건**: 로그인 상태로 대시보드에 접속한 상태  
> **관련 기능**: F-02 (프로필 설정)

### 흐름 A: username(닉네임) 수정

1. 사용자는 **표시 이름을 변경하기 위해** 대시보드의 username 텍스트를 클릭한다.
2. 시스템은 **인라인 편집 모드를 제공하기 위해** 해당 텍스트를 편집 가능한 입력 필드로 전환한다.
3. 사용자는 **새로운 닉네임을 입력하기 위해** 원하는 이름을 입력한다.
4. 사용자는 **변경 사항을 확정하기 위해** 입력 필드 바깥을 클릭하거나 Enter 키를 누른다.
5. 시스템은 **수정 내용을 반영하기 위해** Firestore의 사용자 문서에서 `username` 필드를 업데이트한다.

### 흐름 B: displayName(URL 식별자) 수정

1. 사용자는 **공개 페이지 URL을 변경하기 위해** 대시보드의 displayName 텍스트를 클릭한다.
2. 시스템은 **인라인 편집 모드를 제공하기 위해** 해당 텍스트를 편집 가능한 입력 필드로 전환한다.
3. 사용자는 **새로운 식별자를 입력하기 위해** 영문 소문자, 숫자, 하이픈으로 구성된 값을 입력한다.
4. 시스템은 **중복 여부를 확인하기 위해** 실시간으로 `usernames` 컬렉션에서 중복 검사를 수행한다.
   - 중복인 경우 → "이미 사용 중인 이름입니다" 오류 메시지를 표시한다.
   - 사용 가능한 경우 → "사용 가능합니다" 확인 메시지를 표시한다.
5. 사용자는 **변경 사항을 확정하기 위해** 입력 필드 바깥을 클릭하거나 Enter 키를 누른다.
6. 시스템은 **URL 매핑을 갱신하기 위해** 기존 `usernames/{이전displayName}` 문서를 삭제하고, 새 `usernames/{새displayName}` 문서를 생성하는 트랜잭션을 실행한다.
7. 시스템은 **사용자 정보를 갱신하기 위해** Firestore 사용자 문서의 `displayName` 필드를 업데이트한다.

### 흐름 C: 소개글(Bio) 수정

1. 사용자는 **자기소개를 변경하기 위해** 대시보드의 소개글 텍스트를 클릭한다.
2. 시스템은 **인라인 편집 모드를 제공하기 위해** 해당 텍스트를 편집 가능한 입력 필드로 전환한다.
3. 사용자는 **새로운 소개글을 입력하기 위해** 원하는 내용을 입력한다. (최대 150자)
4. 사용자는 **변경 사항을 확정하기 위해** 입력 필드 바깥을 클릭하거나 Enter 키를 누른다.
5. 시스템은 **수정 내용을 반영하기 위해** Firestore의 사용자 문서에서 `bio` 필드를 업데이트한다.

### 완료 조건
- Firestore의 사용자 문서가 수정된 값으로 업데이트되어 있다.
- (displayName 변경 시) `usernames` 컬렉션의 매핑이 갱신되어 있다.
- 공개 프로필 페이지에 변경 사항이 반영되어 있다.

## S-06. 공개 프로필 페이지 방문

> **사용자 유형**: 방문자  
> **시작 조건**: 소유자가 공유한 마이링크 URL을 전달받은 상태  
> **관련 기능**: F-04 (링크 파비콘 아이콘), F-05 (공개 프로필 페이지), F-06 (반응형 디자인)

### 흐름

1. 방문자는 **소유자의 링크 모음을 확인하기 위해** 공유받은 URL(`도메인/{displayName}`)에 접속한다.
2. 시스템은 **사용자 정보를 불러오기 위해** `usernames/{displayName}` 문서에서 `userId`를 조회한 뒤, 해당 사용자의 프로필과 링크 데이터를 Firestore에서 가져온다.
3. 시스템은 **프로필 정보를 표시하기 위해** 소유자의 프로필 이미지(원형), username(닉네임), 소개글(Bio)을 렌더링한다.
4. 시스템은 **링크 목록을 표시하기 위해** 등록 순서(createdAt)대로 링크 카드를 렌더링하며, 각 카드에 Google Favicon API를 통해 가져온 파비콘 아이콘을 함께 표시한다.
5. 방문자는 **원하는 링크로 이동하기 위해** 관심 있는 링크 카드를 클릭한다.
6. 시스템은 **링크를 열어주기 위해** 해당 URL을 새 탭(target="_blank")으로 연다.

### 완료 조건
- 소유자의 프로필 정보와 링크 목록이 정상적으로 렌더링되어 있다.
- 링크 클릭 시 새 탭에서 해당 페이지가 열린다.
- 모바일/태블릿/데스크탑 기기에서 레이아웃이 정상적으로 표시된다.

## S-07. 존재하지 않는 페이지 방문

> **사용자 유형**: 방문자  
> **시작 조건**: 유효하지 않거나 존재하지 않는 displayName으로 접속 시도  
> **관련 기능**: F-05 (공개 프로필 페이지)

### 흐름

1. 방문자는 **특정 사용자의 페이지를 보기 위해** URL(`도메인/{존재하지않는displayName}`)에 접속한다.
2. 시스템은 **사용자를 조회하기 위해** `usernames/{displayName}` 문서를 검색한다.
3. 시스템은 **해당 사용자가 없음을 알리기 위해** 404 에러 페이지를 표시한다.
4. 방문자는 **서비스를 알아보기 위해** 404 페이지에 안내된 홈 페이지 링크를 클릭하여 랜딩 페이지(`/`)로 이동한다.

### 완료 조건
- 404 페이지가 정상적으로 표시된다.
- 홈 페이지로 돌아갈 수 있는 링크가 제공된다.

## S-08. 로그아웃

> **사용자 유형**: 소유자  
> **시작 조건**: 로그인 상태  
> **관련 기능**: F-01 (Google 소셜 로그인)

### 흐름

1. 사용자는 **세션을 종료하기 위해** 대시보드의 "로그아웃" 버튼을 클릭한다.
2. 시스템은 **인증 세션을 정리하기 위해** Firebase Auth에서 로그아웃 처리를 수행한다.
3. 시스템은 **비인증 상태를 반영하기 위해** 사용자를 로그인 페이지(`/login`) 또는 랜딩 페이지(`/`)로 리다이렉트한다.

### 완료 조건
- Firebase Auth 세션이 종료되어 있다.
- 대시보드 접근 시 로그인 페이지로 리다이렉트된다.

---

## S-09. 재로그인 (기존 사용자)

> **사용자 유형**: 기존 사용자 (소유자)  
> **시작 조건**: 이전에 가입한 적 있으며 현재 로그아웃 상태  
> **관련 기능**: F-01 (Google 소셜 로그인)

### 흐름

1. 사용자는 **대시보드에 다시 접근하기 위해** 로그인 페이지(`/login`)에 접속한다.
2. 사용자는 **본인 인증을 하기 위해** "Google로 로그인" 버튼을 클릭한다.
3. 사용자는 **기존 계정으로 로그인하기 위해** Google OAuth 팝업에서 자신의 Google 계정을 선택한다.
4. 시스템은 **기존 사용자임을 확인하기 위해** Firestore에서 사용자 문서 존재 여부를 조회한다.
5. 시스템은 **기존 데이터를 보여주기 위해** 사용자를 대시보드(`/dashboard`)로 리다이렉트한다.

### 완료 조건
- 이전에 등록한 링크와 프로필 정보가 그대로 유지되어 있다.
